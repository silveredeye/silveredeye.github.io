FAQ

Are you going to do roster change visualisations for all teams?

I hope to do all 24 teams, yes.

Are the charts automatically updated?

Nope. The data is manually compiled for the HTML page, it does not update automatically when there's a roster change on the Blaseball site. I wanted something that would stay up (if not current) even if I couldn't update the code for an unspecified amount of time.

Are you going to note Alternates/being Elsewhere/being Scattered/etc?

Probably not. These charts are already visually busy, I'd like to keep extra information to a minimum.

There's a roster change event missing?

I have left out the cases where the player's statue changes twice during the same game, meaning they end the game in the same place they started (e.g. they Feedbacked twice during the same game, or they Fax Machined twice during the same game). These tiny blips are so tiny that my graph doesn't display them, and they kind of give me a headache.

If that's not the case then I may have made an error. Do let me know if you can.

Can I help? (Even if I don't know how to code.)

Yes! Compiling the data is a bunch of work (especially for teams who were early adopters of Fax Machine and Voicemail), if you compile the data I can code the team's page much faster (and credit you). If I have the data wrong somewhere I would like to know it. And I can always use another set of eyes for stuff like "is this wonky on mobile? do these colors work? I think these should be another sorting option? that's a typo there?".

Can I help? (I do know how to code.)

Yes! Create a pull request. :D
